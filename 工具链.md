# 工具链

本文件按任务场景说明 ChatREQ 科研训练中建议使用的工具链。工具不是目标，工具的作用是帮助学生稳定地产出文献笔记、案例材料、实验记录、原型验证和成果报告。

| 场景 | 工具链 | 用途 | 最低掌握要求 | 产出物 |
| --- | --- | --- | --- | --- |
| 周报与文档 | Markdown + VS Code/Typora + GitHub | 编写周报、任务报告、目录说明和阶段总结 | 会创建 Markdown 文件，会使用标题、列表、表格，会提交到仓库 | `weekly/` 周报、`reports/` 报告、根目录说明文件 |
| 文献管理 | Zotero + PDF 阅读器 | 管理论文、做阅读标注、整理引用信息 | 会导入论文、记录题目作者年份、摘录核心问题和方法 | `reading/` 文献阅读记录 |
| 需求案例整理 | Markdown/Excel + GitHub | 收集用户需求表达，整理意图要素和待澄清问题 | 会按字段整理案例，会区分目标、角色、对象、动作、约束和场景 | `datasets/` 案例集与分析表 |
| 知识条目整理 | Markdown/Excel/JSON + GitHub | 整理需求工程知识、领域知识和项目上下文知识 | 会记录知识来源、知识类型、适用任务和预期作用 | `knowledge/` 知识条目表和映射表 |
| Prompt 实验 | Prompt 记录表 + 实验记录 + GitHub | 管理普通模板、知识增强模板和智能体模板 | 会记录版本、输入、输出、评价结果和失败情况 | `prompts/` 模板，`experiments/` 实验记录 |
| 智能体设计 | 流程图工具 + Markdown | 设计智能体角色、职责、输入输出和协作流程 | 会画基本流程图，会说明每个角色对应的需求工程子任务 | `agents/` 角色表、输入输出说明、协作流程图 |
| 实验记录 | Python/Jupyter/Markdown | 进行案例实验、记录对比结果和计算简单指标 | 会运行基本脚本或 Notebook，会保存输入输出和评价表 | `experiments/` 实验记录，`reports/` 实验报告 |
| 原型验证 | Python + Streamlit/FastAPI 可选 | 实现最小可运行的 ChatREQ 验证模块 | 会运行本地程序，会说明依赖、启动方式、输入输出和截图 | `prototype/` 原型说明、运行说明、截图 |
| 汇报展示 | PPT/WPS + 截图/录屏 | 组会汇报阶段进展、实验结果和原型效果 | 会制作结构清晰的汇报材料，会展示关键证据 | `meetings/` 组会记录，`reports/` 阶段汇报材料 |
| 归档交接 | GitHub + README + archive | 阶段结束时整理材料，便于复查和继续研究 | 会按目录归档，会写清材料清单和用途 | `archive/` 阶段归档包 |

## 1. 文档工具使用要求

所有正式训练材料优先使用 Markdown 编写。Markdown 文件应标题清晰、结构稳定、字段完整。涉及表格的案例、知识条目和实验记录，可以先使用 Excel 整理，再转为 Markdown 或保留表格文件。

## 2. 代码与原型工具使用要求

原型模块可以使用 Python、Streamlit、FastAPI 或其他轻量工具实现。原型不追求复杂页面，而是要能验证知识增强、多智能体协作或需求质量检查机制。

## 3. 实验工具使用要求

实验记录应包含实验问题、输入案例、使用知识、智能体设置、输出结果、评价维度和失败分析。实验结果应能被复查，必要时保留脚本、参数和运行说明。

## 4. GitHub 使用要求

每次提交应尽量对应明确工作内容。提交前检查文件路径、文件名、内容完整性和是否存在临时无关内容。仓库中的关键材料应能通过 README 和目录说明快速找到。
