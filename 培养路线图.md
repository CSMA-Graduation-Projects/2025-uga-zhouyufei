# 培养路线图

本路线图用于指导周雨飞围绕 ChatREQ 方向完成本科科研训练。路线图强调阶段推进、持续沉淀和成果闭环。每个阶段都应形成可检查、可归档、可复用的材料。

## 1. 总体路径

本训练周期按照以下路径推进：

1. 方向理解：理解需求工程、智能需求工程、知识增强和多智能体协作。
2. 案例分析：收集用户需求表达案例，分析意图不完整、模糊、隐含和冲突问题。
3. 失败归纳：分析通用大语言模型在需求工程任务中的失败模式。
4. 知识整理：整理需求工程知识、领域知识和项目上下文知识。
5. 知识增强实验：比较普通生成与知识增强生成的质量差异。
6. 多智能体设计：设计需求访谈、知识注入、需求分析、规约生成、质量检查和归档流程。
7. 质量检查实验：验证需求质量检查机制能否发现问题。
8. 原型模块验证：完成最小可运行模块，支撑研究问题验证。
9. 论文预研整理：形成可支撑论文写作的问题、方法、实验和贡献材料。
10. 专利交底整理：形成技术问题、技术方案、流程步骤和实施例。

## 2. 阶段安排

| 阶段 | 对应任务 | 重点工作 | 主要产物 |
| --- | --- | --- | --- |
| 入门阶段 | M01 | 方向理解、概念整理、基础阅读 | 概念笔记、方向理解报告 |
| 问题发现阶段 | M02-M03 | 收集需求意图案例，分析模型失败现象 | 案例集、失败类型标注表、分析报告 |
| 方法准备阶段 | M04 | 整理知识条目，建立知识与失败类型映射 | 知识条目表、映射表、整理报告 |
| 方法验证阶段 | M05-M07 | 进行知识增强、多智能体设计和质量检查实验 | Prompt 模板、实验记录、角色设计、实验报告 |
| 原型支撑阶段 | M08 | 实现最小可运行原型模块 | 运行说明、示例输入输出、截图、验证报告 |
| 成果沉淀阶段 | M09-M10 | 整理论文预研和专利交底材料 | 论文预研报告、贡献清单、专利交底草稿 |

## 3. 每月推进节奏

每月应至少完成一次阶段复盘。复盘时重点检查：

- 是否有新增可归档材料；
- 是否有明确研究问题支撑；
- 是否完成了案例、知识、实验或原型中的一种实质推进；
- 是否能说明当前工作与论文目标的关系；
- 是否能说明当前工作与专利目标的关系；
- 是否存在长期未解决的问题，需要调整任务范围或方法。

## 4. 阶段转换标准

只有当当前阶段的核心产物达到最低完成标准后，才进入下一阶段。可以并行阅读文献和补充材料，但不能用“继续学习”代替案例、实验、报告和原型产物。

- 从 M01 转入 M02：能够清楚说明 ChatREQ 的研究方向和基本概念。
- 从 M02 转入 M03：已有一定数量的原始需求表达案例和意图分析。
- 从 M03 转入 M04：已归纳出模型在需求工程任务中的主要失败类型。
- 从 M04 转入 M05：已有可用于实验的知识条目和知识映射表。
- 从 M05 转入 M06：已有知识增强对比实验材料。
- 从 M06 转入 M07：已有职责明确的智能体角色和协作流程。
- 从 M07 转入 M08：已有可检验需求质量问题的样例和检查机制。
- 从 M08 转入 M09：已有能够运行并展示研究机制的原型模块。
- 从 M09 转入 M10：已有论文预研材料，可提炼技术问题和技术方案。

## 5. 长期成长目标

本训练的最终目标不是完成若干孤立文档，而是让学生逐步形成科研工作的基本习惯：阅读有问题意识，案例有分析结构，实验有对比条件，原型有验证目标，报告有证据支撑，成果有持续沉淀。
