# knowledge 知识条目目录

本目录用于保存需求工程知识、领域知识和项目上下文知识。知识条目是 ChatREQ 知识增强机制的基础。

## 1. 知识类型

- 需求工程知识：需求获取、分析、规约、验证、管理和质量标准。
- 领域知识：特定业务领域中的角色、流程、规则、数据和约束。
- 项目上下文知识：目标系统范围、已有功能、术语、用户群体和技术限制。

## 2. 知识条目字段

每条知识建议包含：

| 字段 | 说明 |
| --- | --- |
| 知识编号 | 便于引用和追踪 |
| 知识内容 | 清晰表达知识本身 |
| 知识类型 | 需求工程知识、领域知识或项目上下文知识 |
| 知识来源 | 文献、教材、项目文档、人工整理等 |
| 适用任务 | 访谈、分析、生成、检查、归档等 |
| 可解决的失败类型 | 对应 M03 中的失败类型 |
| 预期作用 | 说明该知识如何改善输出 |

## 3. 整理要求

知识条目不能只是概念堆积，应能被用于需求生成或质量检查。每条知识都应说明适用任务和预期作用，并尽量与失败类型建立映射。
