# prototype 原型模块目录

本目录用于保存 ChatREQ 原型模块和验证材料。原型服务于研究验证，不是单纯页面展示。

## 1. 原型目标

原型应至少验证以下机制之一：

- 需求意图要素分析；
- 需求工程知识注入；
- 多智能体协作流程；
- 结构化需求规约生成；
- 需求质量检查；
- 原始意图、知识条目和生成需求之间的追踪。

## 2. 必备材料

- 原型模块说明；
- 运行说明；
- 示例输入输出；
- 系统截图；
- 原型验证报告。

## 3. 质量要求

原型应能够运行，并能说明验证了哪个研究问题。页面美观不是主要目标，关键是研究机制清楚、输入输出可复查、运行过程可说明。
