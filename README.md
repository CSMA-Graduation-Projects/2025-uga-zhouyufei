## 2. 研究方向

ChatREQ 的研究方向为：面向智能需求工程的知识增强与多智能体协作研究。

该方向关注智能需求工程中的真实问题：如何理解利益相关者在自然语言表达中的真实意图，如何澄清模糊、不完整和隐含的需求，如何减少需求生成过程中的意图偏移，如何引入需求工程知识、领域知识和项目上下文提升需求生成质量，以及如何通过多智能体协作完成需求访谈、需求分析、规约生成、质量检查与过程归档。

需求工程的关键不是简单生成需求文本，而是理解利益相关者的真实意图，澄清模糊、不完整和隐含的表达，结合领域知识与需求工程规范，将原始意图转化为结构化、可验证、可追踪的软件需求。

## 3. 科研训练目标

通过本仓库的持续建设，学生应逐步完成以下训练目标：

1. 理解需求工程、智能需求工程、知识增强和多智能体协作的基本思想。
2. 围绕 ChatREQ 方向形成一个清晰、可讨论、可验证的研究问题。
3. 收集并分析一批需求意图案例、失败案例和对比实验记录。
4. 整理需求工程知识条目、智能体角色设计和实验材料。
5. 完成一个最小可运行的 ChatREQ 原型模块或实验性验证模块。
6. 形成可支撑高水平会议或期刊论文的预研材料。
7. 形成面向国家发明专利申请的技术交底草稿。
8. 为后续毕业设计、双创项目、竞赛项目、论文研究或研究生阶段继续深造打下基础。

## 5. 每周更新要求

每周至少完成一次有效更新。有效更新不是简单提交文件名或写一句“继续学习”，而是至少包含以下内容中的若干项：

- 新增或修订周报；
- 新增需求案例、失败案例或实验记录；
- 新增文献阅读笔记或概念整理；
- 新增知识条目、智能体设计或 Prompt 模板；
- 新增实验结果、人工分析或阶段报告；
- 对已有材料进行有依据的修改和补充。

每周周报应放入 `weekly/`，并说明本周对应的任务包、研究问题、完成步骤、新增文件、问题发现、论文支撑、专利支撑和下周计划。

## 6. 阶段成果要求

每个阶段都要有可检查的产物。阶段成果应满足四点要求：

1. 放在仓库中明确路径下；
2. 能说明它支撑了哪个研究问题；
3. 能说明它对论文预研或专利交底的作用；
4. 能被复查、复现或继续补充。

如果只有页面演示，没有问题分析、方法机制和实验验证，不视为高质量科研训练成果。

## 7. 与后续成果的关系

本仓库中的材料将服务于四类后续工作：

- 毕业设计：为选题、需求分析、系统设计、实验评估和论文写作提供长期积累。
- 双创项目：为项目背景、技术路线、创新点和原型演示提供材料。
- 论文预研：为研究动机、问题定义、方法框架、实验材料和评价指标提供证据。
- 专利申请：为技术问题、技术方案、流程步骤、系统组成和实施例提供交底基础。

## 8. 如何开始使用本仓库

建议按以下顺序开始：

1. 阅读 `研究方向说明.md`，理解 ChatREQ 的研究背景。
2. 阅读 `问题定义.md` 和 `解决方案.md`，明确当前训练不是泛泛学习，而是围绕具体研究问题推进。
3. 阅读 `培养路线图.md` 和 `阶段任务清单.md`，了解 M01 至 M10 的整体安排。
4. 从 `tasks/M01-方向理解与基础概念.md` 开始，按任务包要求提交产物。
5. 每周在 `weekly/` 中写周报，并把新增材料放到对应目录。
6. 每月检查一次 `研究闭环检查表.md`，确认已有材料是否能够支撑问题、方法、实验、原型、论文和专利。
