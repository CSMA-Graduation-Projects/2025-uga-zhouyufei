# reading 文献阅读目录

本目录用于保存 ChatREQ 方向相关文献的阅读记录。文献阅读不是简单摘摘要，而是围绕研究问题理解已有工作、发现不足、学习方法和借鉴实验设计。

## 1. 重点文献方向

文献重点包括：

- 需求工程；
- 智能需求工程；
- LLM for Software Engineering；
- 知识增强；
- RAG；
- 多智能体系统；
- 需求质量检查；
- 用户故事生成；
- 需求规约生成。

## 2. 阅读记录字段

每篇文献建议记录：

| 字段 | 说明 |
| --- | --- |
| 题目 | 论文标题 |
| 作者与年份 | 作者、会议或期刊、发表年份 |
| 研究问题 | 论文试图解决什么问题 |
| 方法 | 使用了什么方法或框架 |
| 数据与实验 | 使用了哪些数据、任务和评价指标 |
| 主要结论 | 论文得到什么结果 |
| 局限 | 论文没有解决什么问题 |
| 与 ChatREQ 的关系 | 对本方向有什么启发 |
| 可借鉴内容 | 可借鉴的问题定义、方法、指标或实验设计 |

## 3. 阅读要求

每次阅读应至少形成一份 Markdown 记录。重点不是读完多少篇，而是能把文献内容转化为本仓库中的研究问题、方法设计、实验指标或相关工作材料。
