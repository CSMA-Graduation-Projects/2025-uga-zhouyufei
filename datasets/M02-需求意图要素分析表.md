## 一、选课系统

### "我盯着倒计时读秒，一到九点立刻点击进入选课——然后网页变成了 502。刷新五分钟变成 504。再刷，直接弹回登录页。等终于进去了，所有早八课名额都是 0。"
| 要素 | 分析 |
|------|------|
| 目标 | 在选课开放时成功选上早八课 |
| 角色 | 学生 |
| 对象 | 课程、选课名额 |
| 功能动作 | 选课（提交选课请求） |
| 约束 | 未说明：课容量上限、并发处理策略、排队还是先到先得、是否需要分批次选课？ |
| 场景 | 选课开放瞬间，全校学生同时涌入 |
| 隐含假设 | 系统应该能扛住并发；刷到页面就等于能选上；先到先得是公平的 |
| 待澄清问题 | 你希望系统用排队制（像买火车票一样给你排位）还是先到先得制？如果改成按年级/专业分批次开放选课，你能接受吗？ |

###  "I wanted to select fall and spring, but could not"
| 要素 | 分析 |
|------|------|

| 角色 | 学生 |
| 对象 | 学期、课程 |
| 功能动作 | 多学期课程筛选 |
| 约束 | 未说明：是否允许跨学期选课？系统是按学期独立管理还是关联管理？ |

| 待澄清问题 | ① 你选不同学期的课程是为了对比还是真的两个学期都要选？如果系统自动过滤掉暑期课程，是不是更精准的需求？ |



### "后台跟踪可选择自动选课——当检测到空位时自动尝试选课并通知用户。"

## 二、预约系统

### "必须在预约时间结束的前后二十分钟内签退，不然就算违规。本人非常容易忘事，天天想着不要忘记不要忘记，但每次都是光签到不签退，本周已经因为忘记签退违约三次了……然后系统规定，违约三次，本周就不能再进行预约了。这个设置真的很垃圾。😡"
| 要素 | 分析 |
|------|------|
| 目标 | 正常使用图书馆座位而不因遗忘受到惩罚 |

| 功能动作 | 签到、签退、违约判定 |
| 约束 | 未说明：为什么签退窗口只有前后 20 分钟？违约 3 次封禁 1 周的规则依据是什么？ |

## 三、电商订单系统

### "我在平台上订了一个生日蛋糕，付完钱了。到了那天打开 App 想看看送到哪了，结果发现订单已经被退款了。没有任何人告诉我蛋糕缺货，系统自己就给我退了。我生日派对没有蛋糕，这叫什么事？"

| 场景 | 用户在一个窗口清空购物车，另一个窗口正在结账 |
| 隐含假设 | 不同窗口看到的是同一个购物车状态；系统应该能处理并发操作 |
| 待澄清问题 | ① 你清空购物车是故意的还是误操作？② 如果系统检测到购物车在另一个窗口被修改了，你希望它"暂停结账并提醒你"还是"以最新状态为准"？ |

### "我在平台上买了个快递，物流信息显示'已揽收'之后三天没更新了。我怀疑是不是丢件了，打电话给快递公司说根本没收到这个件。找卖家，卖家说已经发货了让我等。我就卡在中间，钱付了，货不知道在哪，也没人告诉我到底怎么回事。"
| 要素 | 分析 |
|------|------|
| 目标 | 在物流信息异常时获得主动通知和准确的状态查询 |
| 角色 | 买家 |
| 对象 | 物流信息、订单 |
| 功能动作 | 物流状态追踪、异常预警、多方信息同步 |
| 约束 | 未说明：物流信息更新的责任方是谁（平台/卖家/快递公司）？信息不同步时应由谁主动触发查询和协调？ |
| 场景 | 买家付款后物流信息长时间停滞，买卖双方与快递公司三方信息断裂 |
| 隐含假设 | 物流信息应该实时更新；卖家点了"发货"就等于快递已取件；平台有责任协调卖家与快递公司之间的信息不一致 |
| 待澄清问题 | ① 你希望平台在物流信息多长时间没有更新时主动介入——24 小时、48 小时还是 72 小时？② 如果确认丢件，你希望直接补发还是退款？ |

### "我买了一件衣服，到货发现尺码偏小想换大一码。结果系统说只能退货退款不能换货，要我退了重新拍。退了重新拍的时候发现原来的优惠价没了，要多花 30 块。换个尺码而已，为什么这么折腾？"
| 要素 | 分析 |
|------|------|
| 目标 | 换货时保留原订单的优惠价格，且流程简洁不折腾 |
| 角色 | 买家 |
| 对象 | 订单、优惠价格、换货申请 |
| 功能动作 | 换货 + 价格保护 |
| 约束 | 未说明：换货的价格保护策略是什么？哪些优惠类型应保留（满减券/限时折扣/会员价）？部分换货时优惠如何分摊？ |
| 场景 | 用户因尺码/颜色等非质量原因申请换货，但系统只支持退货重购，导致原优惠失效 |
| 隐含假设 | 换货是退货+重新购买的自然替代方案，应享受同等价格待遇；用户不应因合理的换货需求而承担差价损失 |
| 待澄清问题 | ① 你最常在什么情况下需要换货——尺码、颜色、还是轻微质量问题？② 如果换货保留优惠价但需要等更久（等待补货），你能接受吗？ |

## 意图问题类型汇总

| 类型 | 说明 |
|------|------|
| **约束遗漏** | 用户未说明操作的条件、限制和规则  |
| **规则不透明** | 系统有隐式规则但用户不知情，直到被惩罚才发现  |
| **系统替用户做决定** | 系统在未通知用户的情况下自动执行不可逆操作  |
| **多目标混淆** | 用户需求中包含多个互相冲突或优先级不清的目标  |
| **边界缺失** | 异常情况和正常情况的边界未定义  |
| **交互反馈不足** | 系统缺少在关键节点的提示和引导 |
| **执行机制缺失** | 规则有但无法落地执行  |

## 分组理解
1. "用户没说明白"组（约束遗漏、多目标混淆、边界缺失）→ 是用户表达本身的问题
2. "系统没说明白"组（规则不透明、交互反馈不足）→ 是系统设计的问题
3. "系统没做对"组（系统替用户做决定、执行机制缺失）→ 是系统执行的问题
