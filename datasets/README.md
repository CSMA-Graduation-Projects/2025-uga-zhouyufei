# datasets 数据与案例目录

本目录用于保存需求案例、用户意图案例和实验数据。数据材料是 ChatREQ 研究的基础，应保证来源清楚、字段完整、便于复查。

## 1. 可保存的材料

- 用户原始需求表达案例；
- 需求意图要素分析表；
- 需求质量问题样例；
- 实验输入案例；
- 人工标注结果；
- 数据来源说明。

## 2. 案例字段要求

每条需求意图案例建议包含：

| 字段 | 说明 |
| --- | --- |
| 案例编号 | 便于追踪 |
| 原始表达 | 保留用户原始说法 |
| 场景 | 需求发生的业务场景 |
| 目标 | 用户想达到什么目标 |
| 角色 | 涉及哪些用户或系统角色 |
| 对象 | 涉及哪些数据、资源或业务对象 |
| 功能动作 | 需要完成什么操作 |
| 约束 | 时间、权限、规则、性能、合规等限制 |
| 隐含假设 | 用户没有明说但可能默认的前提 |
| 待澄清问题 | 需要进一步询问的问题 |
| 来源 | 案例来源或构造说明 |

## 3. 管理要求

数据文件应避免只保存零散句子。每份数据材料都应说明用途、来源、字段和对应任务包。用于实验的数据应能与实验记录对应起来。
