# agents 智能体设计目录

本目录用于保存 ChatREQ 多智能体角色设计、输入输出说明、协作流程和相关模板。

## 1. 设计原则

多智能体角色不是简单命名，而是对应具体需求工程子任务。每个智能体都应说明它解决什么问题、接收什么输入、产生什么输出、使用哪些知识、与哪些角色协作。

## 2. 建议角色

- Interviewer Agent：负责需求追问与意图澄清；
- Knowledge Agent：负责检索和注入相关知识；
- Analyst Agent：负责识别目标、角色、对象、约束和场景；
- Specification Agent：负责生成结构化需求规约；
- Reviewer Agent：负责需求质量检查；
- Archivist Agent：负责记录意图、知识和需求之间的追踪关系。

## 3. 角色字段

| 字段 | 说明 |
| --- | --- |
| 角色名称 | 智能体名称 |
| 目标 | 该智能体要完成的需求工程任务 |
| 输入 | 接收哪些材料 |
| 输出 | 产生哪些结果 |
| 使用知识 | 需要哪些知识条目 |
| 协作对象 | 与哪些智能体交互 |
| 解决的问题 | 对应哪类需求工程问题 |

## 4. 流程要求

协作流程应能说明从原始需求输入到结构化需求输出，再到质量检查和追踪归档的完整路径。流程图可以使用 Markdown、Mermaid、draw.io 或其他工具完成。
