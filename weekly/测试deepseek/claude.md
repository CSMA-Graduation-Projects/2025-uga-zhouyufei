# 简易库存管理系统（纯前端）软件需求规格说明书

> 文档版本：1.0  
> 日期：2025-03-20  
> 作者：AI

---

## 1. 引言

### 1.1 目的
定义简易库存管理网页的功能性与非功能性需求。系统为纯前端HTML，无后端、无数据库，数据通过浏览器 localStorage 持久化。

### 1.2 范围
支持登录、入库、出库、库存查询、月度统计。所有操作在单页面完成，数据仅存本地，不涉及网络、多用户、报表导出及权限分级。仅适配电脑端，样式简洁。

### 1.3 定义
- **localStorage**：HTML5 客户端存储，数据永久保存。
- **纯前端**：全部代码在浏览器运行，无需服务器。
- **当月**：系统本地时间的自然月。

---

## 2. 总体描述

### 2.1 产品功能
- 登录：预设 admin/123 和 user/123 两个账号。
- 入库：填写物资名称和数量，提交后增加库存并记录日志。
- 出库：选择已有物资，输入数量，库存不足禁止提交，成功扣减库存并记录日志。
- 库存查询：按名称模糊搜索，展示当前库存。
- 数据统计：首页显示当月总入库量和总出库量。

### 2.2 约束
- 纯前端实现（HTML + CSS + JavaScript）。
- 双击 HTML 文件可直接运行，无需任何环境配置。
- 密码明文存储，不提供修改功能。
- 仅适配电脑端（≥1024×768），样式朴素。

### 2.3 假设
- 用户使用支持 localStorage 和 ES6 的现代浏览器（Chrome、Edge、Firefox）。
- 用户不主动清除站点数据则数据持久保存。
- 物资名称 ≤100 字符，单次数量 ≤9999。

## 3. 功能需求

### 3.1 登录模块
| 编号 | 描述 |
|------|------|
| FR-1 | 启动时显示登录页，包含用户名、密码输入框和登录按钮。 |
| FR-2 | 预置 admin/123 和 user/123，无法新增、修改、删除。 |
| FR-3 | 正确账号密码登录后进入主界面，错误则提示。 |
| FR-4 | 登录后显示当前用户名，提供退出按钮。 |

### 3.2 入库模块
| 编号 | 描述 |
|------|------|
| FR-5 | 提供入库操作区域，输入物资名称（必填）和数量（正整数）。 |
| FR-6 | 提交后库存增加（新物资则创建），记录入库日志（名称、数量、时间戳）。 |
| FR-7 | 成功提示并清空输入框，非法输入报错。 |

### 3.3 出库模块
| 编号 | 描述 |
|------|------|
| FR-8 | 提供出库操作区域，支持从已有物资列表中选择。 |
| FR-9 | 输入出库数量，检查库存；不足则提示“库存不足，当前库存：X”，禁止提交。 |
| FR-10 | 成功则扣减库存，记录出库日志，清空输入并提示成功。 |

### 3.4 库存查询
| 编号 | 描述 |
|------|------|
| FR-11 | 提供搜索框，模糊匹配物资名称，实时或点击查询后展示结果。 |
| FR-12 | 结果列表显示物资名称和当前库存，无匹配则显示“暂无”。 |

### 3.5 数据统计
| 编号 | 描述 |
|------|------|
| FR-13 | 首页显示当月总入库量（当月入库数量之和）和总出库量（当月出库数量之和）。 |
| FR-14 | 每次出入库后自动更新，无图表或导出功能。 |

## 4. 非功能需求

### 4.1 性能
- 页面加载 < 3秒，操作响应 < 1秒。
- 支持至少 500 种物资、10,000 条日志。

### 4.2 安全
- 未登录无法访问主界面。
- 账号密码明文，无其他安全措施。
- 禁止通过修改 localStorage 绕过登录。

### 4.3 可用性
- 界面简洁、朴素，系统字体。
- 错误提示清晰，操作反馈即时。
- 布局自适应 1024×768 及以上，无水平滚动条。

### 4.4 兼容性
- 支持 Chrome、Edge、Firefox 最新版。
- 不使用过期或私有特性。

### 4.5 数据持久化
- 数据全部存储于 localStorage，包括库存、入库日志、出库日志、登录状态。
- 关闭页面或浏览器后数据保留，手动清除前不丢失。

## 5. 界面布局建议
- 顶部导航栏：左侧系统名称，右侧用户名及退出按钮。
- 主内容区用卡片划分：入库、出库、库存查询、统计。
- 登录页居中简洁。
